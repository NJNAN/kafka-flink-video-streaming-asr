# 模型对比实验说明

这个功能用于比较不同 ASR 模型、计算类型和并发路数下的性能差异。

## 1. 只生成实验计划

```powershell
python tools/model_ablation.py --dry-run
```

输出：

```text
data/results/model_ablation/model_ablation_report.json
data/results/model_ablation/model_ablation_report.md
```

## 2. 运行真实对比实验

```powershell
python tools/model_ablation.py `
  --models small medium large-v3 `
  --compute-types int8_float16 float16 `
  --streams 1 2 `
  --video-source /videos/input.mp4
```

脚本会按组合重启 ASR/Flink，然后调用现有压测脚本：

```text
tools/benchmark_streamsense.py
```

## 3. 输出指标

| 指标 | 含义 |
| --- | --- |
| 平均延迟 | 字幕从音频切片到 API 写入结果的平均时间 |
| P95 延迟 | 95% 片段低于该延迟 |
| 成功片段 | 成功转写并进入 API 的片段数 |
| 失败片段 | ASR/Flink/API 记录的失败片段 |
| 吞吐 | 每秒处理成功片段数 |

## 4. 答辩说法

```text
我不是随便选择 ASR 模型，而是设计了模型大小、计算精度和并发路数的对比实验。
实时演示可以使用 small/medium，最终高质量字幕可以使用 large-v3。
```
