# 结果数据库说明

项目现在同时使用三类结果存储：

| 存储 | 作用 |
| --- | --- |
| Redis | 实时读取，供 Dashboard 和 API 快速展示 |
| JSONL | 原始运行日志，方便追溯和人工查看 |
| SQLite | 结构化统计，方便查询实验结果 |

SQLite 文件默认位置：

```text
data/results/streamsense.db
```

容器内路径：

```text
/data/results/streamsense.db
```

## 1. 数据表

| 表 | 含义 |
| --- | --- |
| `segments` | 成功字幕片段、延迟、ASR 耗时、retry 次数 |
| `failed_segments` | 失败片段、错误原因、retry 次数 |
| `metrics_samples` | `/api/metrics` 的周期采样结果 |
| `evaluations` | 字幕质量评测结果 |
| `runs` | 预留的任务/会话表 |

## 2. API 查询

```powershell
Invoke-WebRequest http://localhost:8000/api/database/summary
```

按流查询：

```powershell
Invoke-WebRequest "http://localhost:8000/api/database/summary?stream_id=demo-video"
```

## 3. 命令行查询

```powershell
python tools/query_results.py --db data/results/streamsense.db --summary
```

查看某一路：

```powershell
python tools/query_results.py --db data/results/streamsense.db --summary --stream demo-video
```

输出 JSON：

```powershell
python tools/query_results.py --db data/results/streamsense.db --summary --json
```

## 4. 答辩说法

```text
Redis 负责实时读取，JSONL 负责保留原始日志，SQLite 负责结构化实验统计。
这样既能实时展示，也能在实验结束后按 stream_id 查询平均延迟、ASR 耗时和失败情况。
```
