# 自动化验收说明

冒烟测试用于快速检查 Kafka、Flink、ASR、API 和核心 Topic 是否正常。

## 1. 启动系统

```powershell
docker compose up -d --build
```

## 2. 运行冒烟测试

```powershell
python tools/smoke_check.py
```

输出：

```text
data/results/smoke/smoke_report.json
data/results/smoke/smoke_report.md
```

## 3. 检查内容

| 检查项 | 说明 |
| --- | --- |
| API health | `http://localhost:8000/health` |
| ASR health | `http://localhost:8001/health` |
| Flink running job | `http://localhost:8081/jobs/overview` |
| Docker compose services | kafka、redis、api、asr、flink 等服务 |
| Kafka topics | audio-segment、transcription-result、keyword-event、hotword、failed topic |
| API metrics | `/api/metrics` 是否可用 |

## 4. 返回码

- `0`：全部通过。
- `1`：至少一个检查失败。

## 5. 如何理解结果

```text
自动化冒烟测试用于确认 Kafka、Flink、ASR、API 和核心 Topic 都处于可用状态。
如果 6 项检查全部通过，说明系统具备继续运行转写任务的基础环境。
```
