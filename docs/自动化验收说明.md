# 自动化验收说明

本文说明 StreamSense 如何做可重复验收。验收分为三层：后端冒烟测试、数据链路测试、四端入口测试。

## 1. 后端冒烟测试

先启动完整后端：

```powershell
docker compose up -d --build
```

运行冒烟测试：

```powershell
python tools/smoke_check.py
```

输出文件：

```text
data/results/smoke/smoke_report.json
data/results/smoke/smoke_report.md
```

### 检查项

| 检查项 | 检查什么 | 通过代表什么 |
| --- | --- | --- |
| API health | `http://localhost:8000/health` | FastAPI 聚合服务可访问 |
| ASR health | `http://localhost:8001/health` | ASR 服务已启动，模型服务接口可访问 |
| Flink running job | `http://localhost:8081/jobs/overview` | Flink Web UI 可访问，作业状态可查询 |
| Docker compose services | Kafka、Redis、API、ASR、Flink 等容器状态 | 核心容器处于运行状态 |
| Kafka topics | `audio-segment`、`transcription-result`、`keyword-event`、`streamsense.hotword.updates`、`transcription-failed` | 消息队列 Topic 已创建，流处理链路有输入/输出通道 |
| API metrics | `/api/metrics` | 指标接口可读，可以继续做 Dashboard、压测和结果统计 |

### 返回码

| 返回码 | 含义 |
| --- | --- |
| `0` | 全部检查通过，系统具备继续跑转写任务的基础条件 |
| `1` | 至少一个检查失败，需要查看 `smoke_report.md` 或容器日志 |

## 2. 数据链路验收

数据链路关注“输入是否真的经过 Kafka/Flink/ASR/API，而不是只看页面有没有打开”。

| 目标 | 命令/入口 | 预期结果 |
| --- | --- | --- |
| 生成离线字幕 | `python tools/generate_video_subtitles.py --media-path videos/input.mp4 --output-dir data/results/demo` | 输出 SRT/VTT/TXT/JSON 和 report |
| 查询历史统计 | `python tools/query_results.py` | 从 SQLite 读取 stream 统计 |
| 多路并发压测 | `python tools/benchmark_streamsense.py --help` | 可以配置 1/2/4 路输入并生成报告 |
| 在线字幕导出 | `GET /api/streams/{stream_id}/export?format=srt` | 生成可播放字幕文件 |
| 清空演示流 | `DELETE /api/streams/{stream_id}/segments` | 清理内存、Redis 和 JSONL 中对应 stream 的历史 |

## 3. 参数与结果解释

| 参数或指标 | 含义 | 结果怎么理解 |
| --- | --- | --- |
| `stream_id` | 一路输入流的标识，例如 `demo-video`、`desktop-live`、`meetflow-tablet` | 用来区分不同视频、桌面实时流和移动端会议流 |
| `run_id` | 一次会话的标识 | 移动端和实时端用它避免历史片段混入本次结果 |
| `segment_id` | 单个音频片段 ID | 用来追踪某段音频从接入到识别完成的全过程 |
| `end_to_end_time_ms` | 音频片段创建到结果写回的总耗时 | 越低越好，用于衡量实时性 |
| `asr_inference_time_ms` | ASR 模型推理耗时 | 主要受模型大小、设备和音频长度影响 |
| `failed_segments` | 失败片段数 | 越低越好；失败会进入 `transcription-failed` 以便追踪 |
| `recent_throughput_segments_per_second` | 最近采样窗口的片段吞吐 | 用于观察多路并发时系统是否积压 |

## 4. 四端入口验收

| 入口 | 验收命令 | 可观察结果 |
| --- | --- | --- |
| Web Dashboard | `docker compose up -d --build` 后打开 `http://localhost:8000` | 实时字幕列表、关键词、延迟/P95/吞吐卡片、历史曲线、失败片段 |
| 离线 Electron 工作台 | `cd desktop-ui && npm run build` | React/TypeScript/Vite 构建通过；开发运行用 `npm run electron:dev` |
| 实时 Electron 采集端 | `cd desktop-ui-live && npm run build` | 实时端前端构建通过；开发运行用 `npm run electron:dev` |
| MeetFlow 手机/平板 App | `cd meeting-assistant-tablet && npm run build` | 移动端 Web 构建通过；HTTPS 调试用 `npm run dev:https` |
| Android APK | `cd meeting-assistant-tablet && npx cap sync android && .\android\gradlew.bat -p android assembleDebug` | 生成 `com.streamsense.meetflow` debug APK |

## 5. 常见失败定位

| 现象 | 优先检查 |
| --- | --- |
| API health 失败 | `docker compose ps`、`docker logs streamsense-api` |
| ASR health 失败 | GPU/CPU 配置、模型下载、`docker logs streamsense-asr` |
| Flink running job 失败 | Flink JobManager、TaskManager、submitter 日志 |
| Kafka topics 缺失 | `topic-init` 容器是否完成，Kafka/Zookeeper 是否正常 |
| Dashboard 没有新字幕 | `audio-segment` 是否有消息、ASR 是否返回结果、`/api/streams/{stream_id}/segments` 是否有数据 |
| 移动端无法打开麦克风 | 是否使用 HTTPS、浏览器/系统麦克风权限是否允许 |
