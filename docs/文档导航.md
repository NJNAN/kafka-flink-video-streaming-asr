# 项目文档导航

这个项目当前保留了几类不同用途的文档。为了后续答辩、维护和提交 Git 时不混乱，建议按下面顺序阅读。

## 1. 建议阅读顺序

1. [README.md](../README.md)
   先看这个。它负责说明项目是什么、怎么启动、怎么生成字幕、怎么批量验收。

2. [傻瓜式实现文档.md](./傻瓜式实现文档.md)
   如果只是想把项目跑起来，先按这份文档一步一步做。

3. [原理解说.md](./原理解说.md)
   如果要理解为什么用 Kafka、Flink、ASR 服务、VAD、句子缓冲和动态热词，看这份。

4. [新手上手文档.md](./新手上手文档.md)
   如果目标是在 1～2 天内读懂结构、跑起来、改一个小功能，并知道后续怎么深入，看这份工程笔记。

5. [求精v1.md](./求精v1.md)
   如果想继续提高课设完成度，看这份。它按优先级写了质量评测、答辩证据包、监控历史、失败重试、结构化结果库等增强方向。

6. [基于 Kafka-Flink 的视频流语音转写与关键词分析系统_需求文档.md](../基于%20Kafka-Flink%20的视频流语音转写与关键词分析系统_需求文档.md)
   这是原始需求背景，适合答辩时解释“为什么要做这个系统、最初功能边界是什么”。

7. [字幕质量评测说明.md](./字幕质量评测说明.md)
   说明如何用参考文本计算 CER、WER、关键词命中率和补漏缺口。

8. [答辩材料整理说明.md](./答辩材料整理说明.md)
   说明如何一键整理 README、报告、字幕样例和评测结果。

9. [答辩完整讲稿.md](./答辩完整讲稿.md)
   适合答辩前直接照着练，完整覆盖项目背景、架构、实现、求精、实验、演示和问答。

10. [结果数据库说明.md](./结果数据库说明.md)
   说明 Redis、JSONL、SQLite 三类结果存储的分工和查询方法。

11. [领域Profile说明.md](./领域Profile说明.md)
   说明如何为大数据课设、课程视频、会议录音等场景切换热词和纠错表。

12. [自动化验收说明.md](./自动化验收说明.md)
   说明如何运行冒烟测试，快速检查 Kafka、Flink、ASR、API 和 topic。

13. [模型对比实验报告.md](./模型对比实验报告.md)
   说明如何比较不同 ASR 模型、计算类型和并发路数。

14. [优化版课题与实施方案.md](../优化版课题与实施方案.md)
   适合写开题、中期检查或论文“技术路线”部分。

15. [StreamSense_问题解决方案.md](../StreamSense_问题解决方案.md)
   这是问题分析与改进思路参考文档，适合解释早期为什么会出现误识别、断句差、漏字幕等问题。

16. [StreamSense_泛化字幕生成优化说明.md](../StreamSense_泛化字幕生成优化说明.md)
   这是目前最终落地方案的专项说明，重点看泛化能力、补漏机制、批量验收和剩余边界。

17. [Git提交与仓库说明.md](./Git提交与仓库说明.md)
   这是仓库使用文档，说明哪些文件应该提交，哪些文件不能提交。

18. [desktop-ui.md](./desktop-ui.md)
   说明 StreamSense 桌面工作台的 Web/Electron 运行方式。

19. [benchmark.md](./benchmark.md)
   说明单路、2 路、4 路压测脚本、输出报告和指标含义。

20. [system_metrics.md](./system_metrics.md)
   说明链路时间戳、监控 API、多路流查询和动态热词池。

21. [本次性能压测实验报告.md](./本次性能压测实验报告.md)
   记录本次 1 路、2 路、4 路实测数据、指标解释、瓶颈分析和结论。

22. [windows-launcher.md](./windows-launcher.md)
   说明 Windows 桌面启动器、Docker Compose 控制、打包 exe 和常见故障。

## 2. 各文档用途

- `README.md`：项目总入口，面向“第一次接触项目的人”。
- `傻瓜式实现文档`：一步一步运行项目、生成字幕、排查常见问题。
- `原理解说`：用更容易讲清楚的方式解释系统架构和模块分工。
- `新手上手文档`：按工程阅读顺序解释项目结构、调用链、关键函数、调试方法和练手任务。
- `求精v1`：继续提高课设完成度的增强方案和逐步实现路线。
- `字幕质量评测说明`：说明参考文本评测、CER/WER、关键词命中率和评测报告。
- `答辩材料整理说明`：说明如何一键生成课程答辩证据包。
- `答辩完整讲稿`：完整答辩话术，突出项目完整度和技术亮点。
- `结果数据库说明`：说明 SQLite 结构化实验库和查询工具。
- `领域Profile说明`：说明不同领域热词、纠错词表和 `--profile` 参数。
- `自动化验收说明`：说明演示前冒烟测试。
- `模型对比实验报告`：说明模型和并发对比实验。
- `需求文档`：说明课题目标和系统边界。
- `优化版课题与实施方案`：说明课题包装、模块划分和实施路径。
- `问题解决方案`：说明遇到的问题以及参考改进方向。
- `泛化字幕生成优化说明`：说明当前代码实现后的最终方案。
- `Git提交与仓库说明`：说明仓库管理方式。
- `desktop-ui.md`：说明前端工作台和 Electron 桌面模式。
- `benchmark.md`：说明性能压测运行方式和报告字段。
- `system_metrics.md`：说明监控指标、多路流接口和动态热词状态。
- `本次性能压测实验报告.md`：说明本次实验数据、可用结论和后续优化方向。
- `windows-launcher.md`：说明 Windows 启动器交付与故障处理。

## 3. 提交 Git 前先看什么

如果你准备把项目上传到 GitHub 或 Gitee，优先先看：

1. [README.md](../README.md)
2. [Git提交与仓库说明.md](./Git提交与仓库说明.md)
3. [.gitignore](../.gitignore)

这样可以避免把本地模型、测试视频、生成字幕结果和 `.env` 误提交到远程仓库。
