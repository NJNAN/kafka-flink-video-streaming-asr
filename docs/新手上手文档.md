# StreamSense 新手上手文档

这份文档按“新手带教工程笔记”的方式写。目标不是复述 README，而是让你在 1 到 2 天内知道项目解决什么问题、从哪里启动、数据怎么流动、先读哪些代码、怎么改一个小功能。

## 0. 一句话说清楚这个项目

StreamSense 的核心价值是：把真实视频或视频流里的语音，经过 `FFmpeg -> Kafka -> Flink -> 本地 Whisper ASR -> FastAPI/Redis` 处理成实时字幕、关键词事件和可导出的字幕文件。

这个判断来自这些文件：

- `README.md`：项目说明它读取真实视频文件或 RTSP/HTTP 视频流，用本地 `faster-whisper` 转写，并导出 `SRT / VTT / TXT / JSON`。
- `docker-compose.yml`：定义了 `ingest`、`kafka`、`flink-job-submitter`、`asr`、`api`、`redis` 等完整服务。
- `services/ingest/ingest_video.py`：`main()` 读取 `VIDEO_SOURCE`，调用 `run_vad_segment_mode()` 或 `run_fixed_segment_mode()` 产生音频切片。
- `flink/transcription_job.py`：`main()` 组装 Kafka source/sink，`transcribe_segment()` 调用 ASR 服务。
- `services/asr/asr_service.py`：`transcribe()` 和 `transcribe_media()` 调用 `transcribe_with_model()` 做 Whisper 推理。
- `services/api/app.py`：`consume_transcripts()` 消费转写结果，`handle_ready_transcript()` 保存字幕和关键词。

## 1. 典型使用场景

1. 课程项目或答辩演示：用真实视频证明 Kafka、Flink、Redis、ASR、Dashboard 能串成一个流式系统。对应文件是 `docker-compose.yml`、`README.md`、`docs/本次性能压测实验报告.md`。
2. 给本地视频生成字幕：先启动 ASR/API，再运行 `tools/generate_video_subtitles.py`。核心函数是 `resolve_media_path()`、`transcribe_media()`、`build_subtitle_items()`、`recover_uncovered_gaps()`。
3. 实时监控语音转写和关键词：打开 `http://localhost:8000`，静态页面由 `services/api/static/index.html`、`services/api/static/app.js` 和 `services/api/static/styles.css` 提供。
4. 桌面工作台操作：进入 `desktop-ui/` 运行 Electron。核心文件是 `desktop-ui/electron/main.ts`、`desktop-ui/electron/preload.ts`、`desktop-ui/src/App.tsx`、`desktop-ui/src/components/AppShell.tsx`。
5. 多路流压测：运行 `tools/benchmark_streamsense.py`。核心函数是 `start_ingest()`、`wait_until_segments_stable()`、`kafka_lag()`、`build_report()`。

## 2. 技术栈

后端和流处理：

- Docker Compose：`docker-compose.yml` 负责一键启动所有服务。
- Kafka/Zookeeper：`docker-compose.yml` 里的 `kafka`、`zookeeper`、`topic-init`。
- PyFlink：`flink/transcription_job.py`，依赖写在 `flink/requirements.txt`。
- FastAPI：`services/api/app.py` 和 `services/asr/asr_service.py`。
- Redis：`docker-compose.yml` 的 `redis` 服务，API 里 `save_to_redis()` 写入结果。
- faster-whisper：`services/asr/asr_service.py` 的 `ModelHolder.get_model()` 和 `transcribe_with_model()`。
- FFmpeg/FFprobe：`services/ingest/ingest_video.py`、`services/asr/asr_service.py`、`tools/generate_video_subtitles.py` 都会调用。
- WebRTC VAD：`services/ingest/ingest_video.py` 的 `run_vad_segment_mode()`。
- jieba：`services/api/app.py` 的 `extract_keywords()`、`discover_hotword_candidates()`。

前端和桌面端：

- Web Dashboard：`services/api/static/app.js` 定时请求 `/api/status`、`/api/transcripts`、`/api/keywords`。
- React + Vite + TypeScript：`desktop-ui/package.json`、`desktop-ui/src/`。
- Electron：`desktop-ui/electron/main.ts` 负责 Docker Compose 控制、本地文件选择和字幕任务；`desktop-ui/electron/preload.ts` 暴露安全 IPC。

## 4. 程序从哪里启动

项目有 4 种常用启动入口。

### 4.1 完整流式系统入口

命令：

```powershell
docker compose up -d --build
```

调用链：

```text
docker-compose.yml
  -> topic-init 创建 audio-segment / transcription-result / keyword-event / streamsense.hotword.updates
  -> ingest 容器运行 services/ingest/ingest_video.py:main()
      -> wait_for_local_source()
      -> wait_for_kafka()
      -> run_vad_segment_mode() 或 run_fixed_segment_mode()
      -> publish_chunk_message() 写入 Kafka audio-segment
  -> flink-job-submitter 容器运行 flink/transcription_job.py:main()
      -> build_kafka_source()
      -> stream.key_by(...).map(transcribe_segment)
      -> transcribe_segment() 请求 services/asr/asr_service.py 的 /transcribe
      -> build_kafka_sink() 写入 Kafka transcription-result
  -> api 容器运行 services/api/app.py:startup()
      -> load_custom_keywords()
      -> load_asr_corrections()
      -> consume_transcripts()
      -> split_to_sentence_transcripts()
      -> handle_ready_transcript()
      -> save_to_redis() + save_to_file() + publish_keyword_event()
```

### 4.2 ASR 服务入口

`docker-compose.yml` 的 `asr` 服务构建 `services/asr/Dockerfile`，运行 `services/asr/asr_service.py`。

核心调用链：

```text
services/asr/asr_service.py:startup()
  -> ModelHolder.get_model() 预加载 faster-whisper
  -> consume_hotword_updates() 后台接收动态热词

POST /transcribe
  -> transcribe()
  -> should_skip_by_energy()
  -> transcribe_with_model()
  -> update_context()

POST /transcribe-media
  -> transcribe_media()
  -> 可选 extract_clip_to_wav()
  -> transcribe_with_model()
  -> offset_result_timing()
```

### 4.3 离线字幕生成入口

命令：

```powershell
python tools/generate_video_subtitles.py --media-path videos/input.mp4 --output-dir data/results/input --basename input
```

调用链：

```text
tools/generate_video_subtitles.py:main()
  -> resolve_media_path()
  -> transcribe_media() 调 ASR /transcribe-media
  -> 调 API /api/discover-hotwords
  -> build_subtitle_items()
  -> detect_speech()
  -> find_uncovered_speech()
  -> recover_uncovered_gaps()
  -> write_srt() / write_vtt() / write_text()
```

### 4.4 桌面端入口

命令：

```powershell
cd desktop-ui
npm run electron:dev
```

调用链：

```text
desktop-ui/package.json scripts.electron:dev
  -> Vite dev server
  -> electron/tsconfig 编译
  -> Electron 运行 desktop-ui/electron/main.ts
      -> findProjectRoot()
      -> registerIpc()
      -> createWindow()
      -> startServices() 可执行 docker compose up -d --build
      -> createTask()
      -> startTask() 调 python tools/generate_video_subtitles.py
  -> desktop-ui/electron/preload.ts 暴露 window.streamsense
  -> desktop-ui/src/App.tsx 或 components/AppShell.tsx 调 IPC
```

## 5. 核心模块职责

### 5.1 `services/ingest/ingest_video.py`

职责：从真实视频/视频流抽音频、切片、写 Kafka。

重点函数：

- `main()`：读取环境变量，决定走 `vad` 还是 `fixed`。
- `build_ffmpeg_input_command()`：统一构造 FFmpeg 输入命令，决定是否加 `-re` 和 RTSP TCP。
- `run_vad_segment_mode()`：用 WebRTC VAD 按语音停顿切 wav。
- `run_fixed_segment_mode()`：按固定秒数切 wav。
- `build_message()`：构造 Kafka 消息，包括 `segment_id`、`stream_id`、时间戳。
- `publish_chunk_message()`：把音频切片元数据写入 `AUDIO_TOPIC`。

### 5.2 `flink/transcription_job.py`

职责：把 Kafka 中的音频片段消息转成 ASR 结果消息。

重点函数：

- `main()`：创建 Flink 执行环境，设置并行度和 checkpoint。
- `build_kafka_source()`：读取 `AUDIO_TOPIC`，consumer group 是 `flink-asr-transcription`。
- `transcribe_segment()`：解析音频消息，请求 `ASR_URL/transcribe`，补齐链路时间戳。
- `build_kafka_sink()`：写入 `TRANSCRIPT_TOPIC`。
- `stream_id_from_raw()`：按 `stream_id` 分组，保证同一路处理逻辑可控。

注意：`build_kafka_source()` 使用 `KafkaOffsetsInitializer.earliest()`，这能避免首次 ASR 慢导致漏消息，但也会导致历史积压影响延迟指标。`README.md` 和 `docs/本次性能压测实验报告.md` 里已经把这个坑写出来了。

### 5.3 `services/asr/asr_service.py`

职责：封装本地 Whisper 推理，并提供 HTTP 接口。

重点类/函数：

- `TranscribeRequest`：Flink 调 `/transcribe` 的请求体。
- `MediaTranscribeRequest`：离线整段视频转写 `/transcribe-media` 的请求体。
- `SpeechDetectRequest`：漏字幕检测 `/detect-speech` 的请求体。
- `ModelHolder.get_model()`：按 `ASR_MODEL`、`ASR_DEVICE`、`ASR_COMPUTE_TYPE` 加载 faster-whisper。
- `transcribe_with_model()`：真正调用 `model.transcribe()`。
- `should_skip_by_energy()`：按 `ASR_ENERGY_THRESHOLD_DBFS` 过滤静音。
- `should_drop_segment_text()`：过滤低质量、重复、片尾模板幻觉。
- `build_hotwords()`：合并静态热词、动态热词和额外热词。
- `consume_hotword_updates()`：消费 API 广播的动态热词。
- `transcribe()`：流式音频片段转写接口。
- `transcribe_media()`：完整视频/音频转写接口。

### 5.4 `services/api/app.py`

职责：消费 ASR 结果，做句子级聚合、关键词分析、热词学习、Redis/文件持久化和查询接口。

重点函数：

- `startup()`：加载词表、纠错表、动态热词，启动后台消费者。
- `consume_transcripts()`：消费 `TRANSCRIPT_TOPIC`。
- `split_to_sentence_transcripts()`：把短切片合并成自然句字幕。
- `flush_stale_sentence_buffers()`：长时间没有新片段时刷出残留句子。
- `extract_keywords()`：自定义词优先，TextRank 次之，词频兜底。
- `build_keyword_event()`：把字幕转成关键词事件。
- `handle_ready_transcript()`：写入内存、Redis、JSONL、Kafka，并触发热词学习。
- `save_to_redis()`：写入 Redis 列表和有序集合。
- `save_to_file()`：追加写 `data/results/transcripts.jsonl` 和 `keyword_events.jsonl`。
- `maybe_learn_hotwords()`：从转写文本中发现动态热词并广播。
- `build_metrics_payload()`：生成 `/api/metrics` 指标。
- `stream_export()`：按 `stream_id` 导出 `json/srt/vtt/txt`。

### 5.5 `tools/`

职责：绕过实时 Kafka/Flink 链路，直接面向“生成最终字幕”和“压测报告”。

重点文件：

- `tools/generate_video_subtitles.py`：最终字幕生成主脚本，适合新手调字幕质量。
- `tools/export_subtitles.py`：从 `transcripts.jsonl` 导出 `srt/vtt/txt`。
- `tools/batch_generate_subtitles.py`：批量处理视频。
- `tools/benchmark_streamsense.py`：多路流压测。

### 5.6 `desktop-ui/`

职责：提供桌面工作台，包装 Docker Compose、字幕生成任务、结果查看。

重点文件：

- `desktop-ui/electron/main.ts`：主进程，负责 `startServices()`、`checkEnvironment()`、`createTask()`、`startTask()`。
- `desktop-ui/electron/preload.ts`：把 IPC 方法暴露为 `window.streamsense`。
- `desktop-ui/src/App.tsx`：简化字幕生成器界面。
- `desktop-ui/src/components/AppShell.tsx`：完整工作台布局。
- `desktop-ui/src/api/apiClient.ts`：请求后端 `/health`、`/api/status`、`/api/results` 等接口并转换成前端状态。

## 7. 配置文件怎么影响运行

### 7.1 `.env.example`

第一次运行通常复制成 `.env`：

```powershell
Copy-Item .env.example .env
```

关键配置：

- `VIDEO_SOURCE`：`services/ingest/ingest_video.py:main()` 读取它决定输入视频。
- `STREAM_ID`：写入 Kafka 消息，后续 API 用它区分不同视频流。
- `INGEST_SEGMENT_MODE`：决定 `main()` 调 `run_vad_segment_mode()` 还是 `run_fixed_segment_mode()`。
- `INGEST_VAD_*`：影响 `run_vad_segment_mode()` 的切片长度、静音判断和补边。
- `ASR_MODEL`、`ASR_DEVICE`、`ASR_COMPUTE_TYPE`：影响 `ModelHolder.get_model()`。
- `ASR_ENABLE_ENERGY_FILTER`、`ASR_ENERGY_THRESHOLD_DBFS`：影响 `should_skip_by_energy()`。
- `ASR_HALLUCINATION_PATTERNS`：影响 `is_boilerplate_hallucination()`。
- `AUDIO_TOPIC`、`TRANSCRIPT_TOPIC`、`KEYWORD_TOPIC`、`HOTWORD_UPDATE_TOPIC`：影响 Kafka topic。
- `FLINK_JOB_PARALLELISM`：影响 `flink/transcription_job.py:main()` 的并行度。
- `SENTENCE_BUFFER_ENABLED`、`SENTENCE_MAX_CHARS`、`SENTENCE_FLUSH_GAP_MS`：影响 `split_to_sentence_transcripts()` 和 `flush_stale_sentence_buffers()`。
- `HOTWORD_AUTO_DISCOVERY_ENABLED`、`HOTWORD_AUTO_ADD_MIN_COUNT`：影响 `maybe_learn_hotwords()`。

### 7.2 `config/custom_keywords.txt`

被 `services/api/app.py:load_custom_keywords()` 读取。命中的词会优先出现在 `extract_keywords()` 结果里，并且 `jieba.add_word()` 会提高分词稳定性。

### 7.3 `config/asr_corrections.txt`

被 `services/api/app.py:load_asr_corrections()` 和 `tools/export_subtitles.py:load_corrections()` 读取。格式是：

```text
错误词=>正确词
```

### 7.4 `config/profiles/`

当前仓库里有 `config/profiles/dino_keywords.txt` 和 `config/profiles/dino_corrections.txt`。它们是领域 profile 示例。是否自动接入默认流程不确定；需要继续看 `tools/` 或启动脚本有没有 profile 参数。目前我在 `tools/generate_video_subtitles.py` 和 `services/api/app.py` 中看到的是默认读取 `config/custom_keywords.txt` 和 `config/asr_corrections.txt`。

### 7.5 `docker-compose.yml`

它不是普通配置文件，而是完整系统编排文件。比如：

- `topic-init` 明确创建 Kafka topic，避免首次启动 topic 不存在。
- `asr` 挂载 `./models:/models`、`./videos:/videos`、`./data:/data`、`./:/workspace:ro`。
- `api` 挂载 `./config:/config` 和 `./data:/data`。
- `ingest` 挂载 `./videos:/videos` 和 `./data:/data`。
- `flink-job-submitter` 运行 `flink run -py /opt/flink/usrlib/transcription_job.py`。

## 8. 数据/请求/任务怎么流动

### 8.1 实时链路

```text
真实视频文件或 RTSP/HTTP 视频流
  -> services/ingest/ingest_video.py
      FFmpeg 解码音频
      WebRTC VAD 或固定秒数切 wav
      data/audio/*.wav
      Kafka audio-segment 消息
  -> flink/transcription_job.py
      KafkaSource 读 audio-segment
      transcribe_segment() HTTP POST /transcribe
  -> services/asr/asr_service.py
      should_skip_by_energy()
      transcribe_with_model()
      返回文本、segments、模型耗时、置信度
  -> Kafka transcription-result
  -> services/api/app.py
      consume_transcripts()
      split_to_sentence_transcripts()
      build_keyword_event()
      save_to_redis()
      save_to_file()
      maybe_learn_hotwords()
  -> Redis + data/results/transcripts.jsonl + keyword_events.jsonl
  -> Web Dashboard / Desktop UI / export API
```

### 8.2 离线最终字幕链路

```text
python tools/generate_video_subtitles.py
  -> /transcribe-media 让 ASR 直接转完整媒体
  -> /api/discover-hotwords 获取候选热词
  -> build_subtitle_items() 生成字幕条目
  -> /detect-speech 查明显有声区间
  -> recover_uncovered_gaps() 对漏段补转写
  -> 输出 .srt / .vtt / .txt / *_report.json / *_final_segments.json
```

### 8.3 桌面任务链路

```text
Electron UI 选择视频
  -> desktop-ui/electron/main.ts:createTask()
      可选 copyVideoToWorkspace()
      写入 data/results/tasks.json
  -> startTask()
      spawn("python", ["tools/generate_video_subtitles.py", ...])
  -> 监听 stdout/stderr 更新任务状态
  -> 生成结果后写 task.outputs 和 task.report_path
  -> React 前端通过 window.streamsense.onTaskUpdate() 刷新 UI
```

## 11. 最容易踩坑的地方

1. Docker Desktop 没启动：`docker compose ps` 会失败，Electron 的 `checkEnvironment()` 也会报 Docker daemon 不可连接。
2. 没有 `videos/input.mp4`：`services/ingest/ingest_video.py:wait_for_local_source()` 会一直等文件。
3. GPU/显存不够：`ASR_MODEL=large-v3`、`ASR_DEVICE=cuda`、`ASR_COMPUTE_TYPE=float16` 对机器要求高。可以改成 `ASR_MODEL=medium` 或 `ASR_DEVICE=cpu`。
4. 首次运行很慢：`ModelHolder.get_model()` 首次会下载/加载模型到 `models/`。
5. 历史 Kafka 积压影响延迟：`flink/transcription_job.py:build_kafka_source()` 从 `earliest` 开始消费，旧消息会拉高端到端延迟。
6. 字幕断句不自然：优先调 `.env` 的 `INGEST_VAD_TARGET_CHUNK_MS`、`INGEST_VAD_HARD_MAX_CHUNK_MS`、`SENTENCE_MAX_CHARS`、`SENTENCE_FLUSH_GAP_MS`。
7. 字幕错词：先改 `config/asr_corrections.txt`，因为 `clean_transcript_text()` 和导出脚本都会应用纠错。
8. 自定义词没生效：确认词是否写在 `config/custom_keywords.txt`，API 启动时 `load_custom_keywords()` 才会加载；改完要重启 API 容器。
9. 端口冲突：`desktop-ui/electron/main.ts` 的 `REQUIRED_PORTS` 列出了 8000、8001、8081、5173、6379、9092、29092。
10. 把大文件提交到 Git：`.gitignore` 已忽略 `models/`、`videos/`、`data/audio/`、`data/results/`、根目录 `input*.mp4`，提交前仍要 `git status --short`。
