# StreamSense 傻瓜式实现文档

这份文档按“照着做就能跑”的方式写。不要先研究源码，先把项目跑起来，再回头看原理。

## 1. 你要先准备什么

电脑上需要有：

1. Windows 10 或 Windows 11。
2. Docker Desktop。
3. Python 3.10 或以上。
4. 一个带声音的视频文件，例如 `input.mp4`。
5. 如果要开发桌面端，再安装 Node.js。

检查命令：

```powershell
docker --version
docker compose version
python --version
```

如果 Docker 命令报错，先打开 Docker Desktop，等它启动完成。

## 2. 进入项目目录

打开 PowerShell，进入项目根目录：

```powershell
cd "D:\作业\大数据\基于 Kafka-Flink 的视频流语音转写与关键词分析系统"
```

确认目录正确：

```powershell
Get-ChildItem
```

应该能看到：

```text
docker-compose.yml
README.md
services
flink
tools
config
videos
```

## 3. 放入测试视频

把你的视频复制到：

```text
videos/input.mp4
```

也可以用 PowerShell 复制：

```powershell
Copy-Item "D:\你的视频.mp4" "videos\input.mp4"
```

检查视频是否存在：

```powershell
Test-Path "videos\input.mp4"
```

输出 `True` 才能继续。

## 4. 创建配置文件

第一次运行需要从模板复制 `.env`：

```powershell
Copy-Item .env.example .env
```

如果提示文件已经存在，说明之前已经创建过，可以不用重复复制。

默认配置会读取容器里的：

```text
/videos/input.mp4
```

这个路径对应 Windows 项目目录下的：

```text
videos/input.mp4
```

## 5. 启动完整系统

在项目根目录执行：

```powershell
docker compose up -d --build
```

第一次启动会比较慢，因为 Docker 要构建镜像，ASR 服务还可能下载 Whisper 模型。

查看容器状态：

```powershell
docker compose ps
```

正常情况下能看到这些服务：

```text
streamsense-zookeeper
streamsense-kafka
streamsense-topic-init
streamsense-redis
streamsense-asr
streamsense-api
streamsense-flink-jobmanager
streamsense-flink-taskmanager
streamsense-flink-job-submitter
streamsense-ingest
```

`topic-init` 和 `ingest` 运行完退出是正常的，不一定一直保持 running。

## 6. 打开网页检查

打开浏览器访问：

```text
http://localhost:8000
```

这是系统 Dashboard。

再访问：

```text
http://localhost:8001/health
```

如果 ASR 正常，会看到类似 JSON 的健康状态。

再访问：

```text
http://localhost:8081
```

这是 Flink Web UI，可以看流处理作业。

## 7. 看日志

如果页面没有结果，先看日志：

```powershell
docker compose logs -f --tail=200
```

只看 ASR：

```powershell
docker compose logs -f --tail=200 asr
```

只看视频接入：

```powershell
docker compose logs -f --tail=200 ingest
```

只看 API：

```powershell
docker compose logs -f --tail=200 api
```

## 8. 生成最终字幕文件

推荐用字幕脚本生成最终可提交的字幕文件：

```powershell
python tools/generate_video_subtitles.py --media-path videos/input.mp4 --output-dir data/results/input --basename input
```

运行完成后查看：

```powershell
Get-ChildItem data\results\input
```

一般会生成：

```text
input.srt
input.vtt
input_subtitle.txt
input_final_segments.json
input_report.json
input_hotwords.json
```

最常用的是：

```text
data/results/input/input.srt
data/results/input/input_subtitle.txt
```

## 9. 处理其他视频

假设你的视频叫：

```text

```

运行：

```powershell
python tools/generate_video_subtitles.py --media-path videos/lecture.mp4 --output-dir data/results/lecture --basename lecture
```

输出会在：

```text
data/results/lecture
```

把多个视频放进 `videos/`，然后执行：

```powershell

```

批量结果在：

```text

```


## 13. 最常见问题

### 13.1 docker compose up 报错连接不上 Docker

原因：Docker Desktop 没启动。

处理：

```powershell
docker compose ps
```

如果还是报错，打开 Docker Desktop，等它启动完成。

### 13.2 ASR 一直很慢

第一次会下载模型，这是正常的。

如果显存不够，编辑 `.env`：

```text
ASR_MODEL=medium
ASR_COMPUTE_TYPE=int8_float16
```

没有 GPU 就改成：

```text
ASR_DEVICE=cpu
ASR_COMPUTE_TYPE=int8
```

改完后重启：

```powershell
docker compose down
docker compose up -d --build
```

### 13.3 没有字幕输出

按顺序检查：

```powershell
Test-Path videos\input.mp4
docker compose ps
docker compose logs --tail=200 ingest
docker compose logs --tail=200 asr
docker compose logs --tail=200 api
```

重点看视频是否有声音、ASR 是否启动、Flink 是否提交成功。

### 13.4 网页显示 Mock 数据

说明前端没有连上真实后端。

先检查：

```text
http://localhost:8000
```

如果打不开，先启动后端：

```powershell
docker compose up -d --build
```

### 13.5 不想把大文件提交到 Git

提交前执行：

```powershell
git status --short
```

确认没有出现：

```text
models/
videos/*.mp4
data/results/
data/audio/
desktop-ui/release/
desktop-ui/release-fresh/
*.exe
```

这些都应该留在本机，不要提交。

## 14. 最小验收清单

项目算跑通，需要满足：

1. `docker compose ps` 能看到核心服务。
2. `http://localhost:8000` 能打开 Dashboard。
3. `http://localhost:8001/health` 能访问。
4. `python tools/generate_video_subtitles.py ...` 能生成 `.srt`。
5. `data/results/...` 下能看到字幕和报告。
6. `git status --short` 不包含视频、模型、打包 exe、运行结果。
