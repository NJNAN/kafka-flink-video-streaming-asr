# 答辩材料整理说明

这个功能用于把项目文档、压测结果、质量评测结果和字幕样例整理到一个目录，方便提交和答辩展示。

## 1. 运行命令

```powershell
python tools/make_defense_package.py `
  --subtitle-dir data/results/single_test `
  --benchmark-dir data/results/benchmark `
  --evaluation-dir data/results/evaluation `
  --output-dir data/results/defense_package
```

## 2. 输出目录

```text
data/results/defense_package/
├── 00_答辩材料清单.md
├── 01_README.md
├── 02_系统原理解说.md
├── 03_运行步骤.md
├── 04_性能压测报告.md
├── 05_字幕质量评测报告.md
├── sample_subtitles/
├── benchmark/
└── evaluation/
```

其中 `00_答辩材料清单.md` 是总入口。

## 3. 注意事项

- 这个目录属于运行结果，默认不建议提交 Git。
- 如果某些报告还没生成，脚本会在 `manifest.json` 和材料清单里标记缺失。
- 最终提交课程材料时，可以直接从这个目录挑选 Markdown、字幕文件和截图。

## 4. 答辩说法

```text
我给项目加了一键材料整理脚本。
它会自动收集 README、原理解说、运行步骤、压测报告、字幕质量评测和字幕样例，
避免最终提交材料分散在不同目录。
```
