# Codex 工作规则

本项目需要 Codex 直接做事，不要只给建议。

## 默认行为

- 用户提出修改、修复、整理、实现、排查、更新文档、准备提交等请求时，默认直接执行。
- 不要停在方案、建议、可能原因、下一步提示上，除非用户明确要求只解释。
- 开始前先快速读取相关文件，确认项目结构和现有写法。
- 文件能改就改，命令能跑就跑，结果能验证就验证。

## 长任务要求

- 任务超过 3 步时，先写一个简短 TODO。
- 每完成一项就更新状态，不要最后一次性标完成。
- 不要因为任务较大就只完成最容易的一小部分。
- 如果遇到错误，先自己排查至少 2 轮，再向用户说明 blocker。

## 代码和文档

- 保持本项目已有风格，不做无关重构。
- 修改代码后，运行能运行的最小验证命令。
- 修改文档后，检查路径、命令、文件名是否真实存在。
- 不要伪造运行结果，不要声称没有验证过的内容已经验证。

## 最终回复

最终回复必须简短说明：

- 实际修改了哪些文件。
- 运行了哪些验证命令。
- 是否还有未解决风险。

如果没有实际修改或验证，必须明确说明原因。
